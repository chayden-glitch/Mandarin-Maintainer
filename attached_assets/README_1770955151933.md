# Mandarin App

A Streamlit app for reading Chinese news with vocabulary highlighting and spaced‑repetition review.

## Quickstart (copy/paste)

### 1) Go to the project folder

```
cd "/Users/ChrisHayden/Documents/Mandarin App"
```

### 2) Run the app

```
streamlit run app.py --server.port 8501
```

### 3) Open in browser

```
http://localhost:8501
```

### 4) Stop the app

```
Ctrl+C
```

## Common daily commands

### App

- Start: `streamlit run app.py --server.port 8501` — launches the app locally so you can view it in a browser.
- Stop: `Ctrl+C` — stops the running app in that terminal.
- Restart: stop, then run the start command again — needed after code or config changes.

## Notes

- If port 8501 is busy, pick another port (e.g., 8502):
  `streamlit run app.py --server.port 8502`

### Git (GitHub)

Git is the tool that tracks changes to the code. GitHub is the shared website where the team’s code lives. You make changes locally, then “commit” them (save a snapshot), and “push” them to GitHub so others can see and review.

Typical flow (in order):
- `git status` → see what changed
- `git add .` → stage those changes
- `git commit -m "message"` → save a snapshot with a description
- `git push` → upload to GitHub

- Check changes: `git status` — shows which files you’ve edited but not saved to Git.
- Stage changes: `git add .` — selects all changes to include in the next commit.
- Commit: `git commit -m "message"` — creates a named snapshot of your staged changes.
- Push: `git push` — uploads your commits to GitHub so others can see them.
- Pull latest: `git pull` — downloads the newest changes from GitHub to your laptop.
- Create branch: `git checkout -b feature/name` — starts a safe, isolated line of work.
- Switch branch: `git checkout branch-name` — moves between lines of work.
