# Chinese Tutor - Mandarin Language Maintenance App

## Overview
A personal Mandarin Chinese language maintenance application designed to help advanced learners retain and expand their vocabulary through spaced repetition. Uses simplified Chinese only. Currently in Phase 1: Corpus & Database with FSRS backend complete.

## Current State
Phase 1 complete with daily review functionality, Phase 2 (News Reader) in progress:
- SQLite database for vocabulary storage
- CSV upload functionality using pandas with full preview before import
- Duplicate detection: checks Chinese characters only, skips duplicates and shows list
- Editable vocabulary table for inline corrections
- Glassmorphism UI with sidebar navigation (Home, Explore, Library, Settings)
- FSRS backend with Recognition/Production card pairs and sibling burial
- Daily review flashcard interface with session tracking and rating buttons
- Practice mode: review random cards when none are due (still updates FSRS)
- Max new cards per day limit (default 5, user-adjustable)
- RSS News Reader with vocabulary highlighting (BBC Chinese, VOA Chinese for in-app reading; NYT/WSJ for external links)
- Gemini-powered pinyin and English translations for article titles
- Source badges and free source prioritization (BBC/VOA appear first)
- Improved article reader with better formatting and expandable vocabulary list

## Project Architecture

### Files
- `app.py` - Main Streamlit application with UI for all tabs
- `database.py` - SQLite database module with vocabulary and cards schemas
- `fsrs_engine.py` - FSRS spaced repetition engine with scheduling and sibling burial
- `rss_reader.py` - RSS feed aggregation, article fetching, and vocabulary highlighting
- `chinese_tutor.db` - SQLite database file (created on first run)

### Database Schema
The `vocabulary` table contains:
- `id` - Primary key
- `simplified` - Chinese characters (simplified)
- `pinyin` - Pronunciation
- `english` - English translation
- `lesson_number` - Optional lesson organization
- `example_sentence` - Optional example sentence
- `buried` - 0 or 1 flag indicating if vocabulary is hidden from reviews

The `cards` table contains:
- `id` - Primary key
- `vocabulary_id` - Foreign key to vocabulary
- `card_type` - 'Recognition' (front: Chinese) or 'Production' (front: English)
- FSRS fields: `stability`, `difficulty`, `elapsed_days`, `scheduled_days`, `reps`, `state`, `last_review`, `due`

### FSRS Engine Functions
- `sync_cards()` - Creates Recognition + Production cards for all vocabulary
- `update_card(card_id, rating)` - Updates card with FSRS scheduling (1=Again, 2=Hard, 3=Good, 4=Easy)
- `get_due_cards()` - Returns due cards with sibling burial (24-hour exclusion), new card limit, and filters out buried vocabulary
- `get_due_cards_count()` - Returns count of due cards (respects new card limit, excludes buried)
- `get_cards_reviewed_today()` - Returns today's review count
- `get_new_cards_introduced_today()` - Returns count of new cards introduced today
- `get_max_new_cards_per_day()` / `set_max_new_cards_per_day()` - Manage new card limit setting
- `bury_card_vocabulary(card_id)` - Buries the vocabulary associated with a card (hides from all reviews)

### Dependencies
- streamlit - Web UI framework
- pandas - CSV parsing
- fsrs - Spaced repetition algorithm
- sqlite3 - Database (built-in)
- feedparser - RSS feed parsing
- newspaper4k - Article text extraction
- jieba - Chinese word segmentation
- edge-tts - Chinese text-to-speech (Microsoft Edge neural voices)

### RSS Reader Functions
- `fetch_all_feeds()` - Aggregates articles from all configured RSS sources
- `fetch_article_content(url)` - Extracts full article text using newspaper4k
- `get_vocabulary_set()` - Returns set of all vocabulary simplified characters
- `highlight_vocabulary(text, vocab)` - Returns text with vocab words highlighted in orange
- `get_vocabulary_matches(text, vocab)` - Returns list of vocabulary words found in text
